[01:51:21] RAG vs Fine-tuning
那因為上週有同學有提到一個問題是說 RAG 跟 Fine-tuning 到底有什麼差別。就是我們老師在上週介紹 RAG 跟 Fine-tuning。
Fine-tuning (微調) 就是做模型的微調，也就是說它用額外的資料重新的訓練這個模型，讓它模型可以內化新知識或特定的風格。
RAG (檢索增強生成) 就像剛剛講的就是它是檢索外部的知識。
所以一個是針對模型來做額外的訓練，一個是直接檢索外部、創一個向量資料庫讓它做額外的檢索。所以這兩個有點不太一樣。

[01:51:55] 那我們為什麼比較少做 Fine-tuning？因為 Fine-tuning 通常會有很高的硬體的需求。Fine-tuning 其實還是有應用場景的，比如說「客服語氣的調整」。比如說你今天是國泰的客服好了，那你特別希望你的客服能回答使用者的問題的時候呢，特別的親切，都講「請、你好、謝謝」，假設你的需求這樣。那你就很適合用來做模型的微調，讓它特別針對你的語氣來做調整。

[01:52:33] 但如果你只是要讓它看懂你的某些的文件，然後來做回答，像是企業內部知識庫啊、文件的回答等等的，那其實你用 RAG 就相當的夠。所以 Fine-tuning 主要的缺點就是它訓練非常的耗時，而且它需要的這個硬體需求相當的大，也是很貴的意思，而且更新並不是這麼的靈活。像 RAG 的話，你就只是要更新文件、更新向量資料庫，你就可以得到一個有點就是得到一個全新的應用場景，這樣子更加靈活一點。

[01:53:11] RAG 跟 Fine-tuning 的差別，我們舉一個小小的例子。就比如説你想要更懂台股跟美股的財報的語境。那我們舉 Llama 3 70B 這個模型。70B 老師上次有講嘛，70B 代表 Billion，是它的參數的意思，也就是說這個模型主要有 700 億個參數。所以你要在你本地端的電腦跑這個模型就是一個很大的問題。

[01:53:37] 像他說要八張這個 NVIDIA A100 的 GPU 才可以跑得動，就是一個相當大的問題。那所以而且你用額外的雲端資源的話，也是有相當大的成本需要做考量。但如果你是使用 RAG 系統的話，你其實只要一張顯卡或甚至是 CPU 就可以來做 RAG 系統的建置跟使用這些模型。

[01:54:04] Metadata (元資料)
那關於今天 Metadata 的介紹。Metadata 也就是描述資料的資料，也就是說它是關於這份內容的背景資訊。比如說你現在有個文件是在講說台積電第二季的年營收它增加多少多少。好，這是一份 PDF 文件好了。有了 Metadata 的幫助以後，你就可以標註這份文件。你可以儲存各種你想要的資訊嘛，比如說 Title，它的文件標題是「台積電第二季營收」；公司 (Company) 是 TSMC；語言 (Language) 是中文；時間 (Date) 等等的。這些透過 Metadata 來做紀錄。

[01:54:47] 所以回到剛剛那個 RAG 這邊。它有點像在 Embedding 的時候，也就是把文件轉成向量存進向量資料庫的時候，把每一個文件都加了一個有點像「行李吊牌」的東西。Metadata 就是這個行李吊牌。

[01:55:04] 那我們加這些行李吊牌以後，像我們把自己的行李加成你的名字，假設是我，我劉子俊，然後他是來自政治大學，他現在讀碩二這樣子。然後你們會發現作業有一個地方是你轉成向量值，你還會先切那個 Chunk。那老師當初設定的是 500 個字一個 Chunk，然後上下會重複 100。也就是說你會把每一個文件再按照 500 個字、500 個字來做切分。

[01:55:30] 那你之後 Metadata 的功用就是：它會切成一個一個片段，但是這些片段、同一份文件的片段都會帶有這份文件的 Metadata。也就是說，以我們剛剛那個行李的例子，雖然說不會有人把行李切分啊。假設你現在有個行李，你上面寫了你的名字，然後你來自哪個國家，你現在的年紀好了。那你把它切成三份，這三份都會有同樣的行李吊牌。所以他不管哪一份以後，他都可以知道「哦，這是來自劉子俊的行李，他是來自中華民國，然後他現在 23 歲」等等。

[02:15:40] 我那時候的回答是這個樣子：
第一，我會先印出檢索到的 Chunk 的原文有哪些。假如原文的這些真的是偏比較不相關的，那我們是不是就考慮我們當初在檢索的時候，Cosine Similarity 的比例是不是要做一些更動？或者說我們是不是要做一些 Rerank（重新排序）的動作，用 Rerank 模型來做這個部分。
第二個，假如我今天發生我的 Chunk 都是相關的，但是語言模型卻回答的不是很好，那我就會朝 Prompt 的方向去改進。看能不能把我們的 Prompt 寫得更詳盡，或是給他更多的人設以及給他一些例子。比如説你想要條列式的列出這份根據——每個 RAG 的那個系統不太一樣嘛，檢索到資料不太一樣——假設是財報的話，你希望能在使用者問問題時是用條列式的方式呈現給來做回答，那你可能就要加在 Prompt 裡面寫的更詳細一點，或甚至是換一個模型來做試看這樣子。
